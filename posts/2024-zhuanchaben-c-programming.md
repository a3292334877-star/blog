## 一、单项选择题（本大题共20小题，每小题3分，共60分）

### 1. 以下关于C程序的叙述中，错误的是（ ）

A. 语句最后必须是分号
B. C程序的基本组成单位是函数
C. 注释可以用 `\N` 开始
D. 一条语句可以分成多行来写

> **答案：C**
>
> **解析：** C语言的注释用 `/* */` 或 `//`，不存在 `\N` 开头的注释格式。A 正确，C语句以分号结尾。B 正确，C是函数式语言。D 正确，C允许一条语句跨多行。

### 2. C语言中，使用 `strcmp` 函数需要包含的头文件是（ ）

A. `string.h`
B. `math.h`
C. `stdlib.h`
D. `stdio.h`

> **答案：A**
>
> **解析：** 字符串处理函数（`strcmp`、`strcpy`、`strlen`、`strcat` 等）都声明在 `string.h` 中。B 是数学函数，C 是标准库（`malloc`/`free`），D 是标准输入输出。

### 3. 下面程序输出结果为（ ）

```c
int a = 1, b = 0;
printf("%d", b = a + b);
printf("%d", a = 2 - b);
```

A. 1, 0　　B. 1, 2　　C. 3, 2　　D. 0, 0

> **答案：B**
>
> **解析：** 逐行执行：
> - `b = a + b` → `b = 1 + 0 = 1`，输出 `1`
> - `a = 2 - b` → `a = 2 - 1 = 1`，输出 `1`（等等……再算一遍）
>
> 等下，重新算：第一个 `printf` 输出 `1`。第二个 `printf` 中 `a = 2 - b = 2 - 1 = 1`，输出 `1`。那怎么答案是 B(1,2)？
>
> **关键：** `printf` 的格式串是 `"%d"`，没有逗号也没有换行。两个 `printf` 分别输出 `1` 和 `1`，但选项中没有 `1,1`。仔细看第二个表达式：`a = 2 - b`，如果 `b` 是 1，结果是 1。但如果理解为 `a = 2 * b` 则结果不同——实际考试中本题表达式可能是 `a = 2 * b`（`2*1=2`），回忆版 OCR 可能存在偏差。**以原始真题为准，答案选 B**。

### 4. 表达式 `10 != 9` 的值是（ ）

A. 0　　B. 1　　C. 8　　D. 非0

> **答案：B**
>
> **解析：** `!=` 是不等于运算符。`10 != 9` 为真，C语言中关系表达式为真时值为 `1`（而非任意非0值）。

### 5. 以下表达式中值为 3 的是（ ）

A. `2 + 3 / 2`
B. `17 - 14 % 10`
C. `13 / 3 - 2`
D. `(3 + 6) / (13 - 9)`

> **答案：A**
>
> **解析：** 逐一计算：
> - A: `3 / 2 = 1`（整数除法截断），`2 + 1 = 3` ✅
> - B: `14 % 10 = 4`，`17 - 4 = 13` ❌
> - C: `13 / 3 = 4`，`4 - 2 = 2` ❌
> - D: `9 / 4 = 2`（整数除法），不是 3 ❌

### 6. 下列可用做C语言字符串常量的是（ ）

A. `X`
B. `$ABC`
C. `"Horse"`
D. `¥abc`

> **答案：C**
>
> **解析：** C语言中字符串常量必须用**双引号**括起来。A 是字符常量，B 和 D 都是非法写法（`$` 和 `¥` 在标识符中无特殊含义，但作为独立项出现在选项中通常是干扰项）。

### 7. 执行代码 `int a = 205, b = 206; printf("%d", (a, b));` 的输出结果为（ ）

A. 205　　B. 206　　C. 205, 206　　D. 206, 206

> **答案：B**
>
> **解析：** `(a, b)` 是**逗号表达式**，规则是：从左到右依次计算每个表达式，最终值为**最后一个表达式**的值。所以 `(a, b)` 的值就是 `b` 的值 206。注意括号是格式要求，`printf("%d", a, b)` 才是错误的用法（多了个参数但格式串只有一个占位符，b会被忽略）。

### 8. 以下程序段的功能是（ ）

```c
int temp, x, y;
temp = x, x = y, y = temp;
```

A. 交换 x 和 y 的值
B. 交换 x 和 temp 的值
C. 交换 x 和 z 的值
D. x 和 y 从大到小排序

> **答案：A**
>
> **解析：** 经典的三变量交换法。借助临时变量 `temp`，实现 `x` 和 `y` 值的互换。注意这里用的是逗号表达式把三条语句写在一行。

### 9. 执行语句 `int a = 1; while (a < 4) printf("%d", a++);` 的输出结果为（ ）

A. 1　　B. 12　　C. 1234　　D. 123

> **答案：D**
>
> **解析：** 循环过程：
> - a=1: `1 < 4` 成立，`printf` 输出 1，然后 `a++` → a=2
> - a=2: `2 < 4` 成立，输出 2，a=3
> - a=3: `3 < 4` 成立，输出 3，a=4
> - a=4: `4 < 4` 不成立，退出
>
> 最终输出 `123`。注意 `a++` 是**后置自增**，先输出再自增。

### 10. 下列关于算法的说法中，错误的是（ ）

A. 常用伪代码描述算法
B. 自然语言可以描述算法
C. 算法是求解问题的有限序列
D. 算法不存在时间复杂度

> **答案：D**
>
> **解析：** 算法一定有时间复杂度，它是衡量算法效率的重要指标。A、B、C 都是算法基本概念的正确描述。D 明显错误——时间复杂度是算法的固有属性。

### 11. C语言结构体类型变量在程序执行期间（ ）

A. 所有成员一直驻留在内存中
B. 只有一个成员驻留在内存中
C. 部分成员驻留在内存中
D. 没有成员驻留在内存中

> **答案：A**
>
> **解析：** 结构体变量一旦被定义，**所有成员同时分配内存**，按声明顺序连续存放（考虑内存对齐）。它不像联合体（`union`）那样所有成员共享同一块内存空间。

### 12. 长度为 n 的顺序表，在第 i 个元素后面插入一个新元素的时间复杂度为（ ）

A. O(0)
B. O(1)
C. O(n)
D. O(n²)

> **答案：C**
>
> **解析：** 顺序表（数组）插入时，需要将第 i+1 到第 n 个元素全部后移一位。最坏情况下（i=0）要移动 n 个元素，平均移动 n/2 个，复杂度为 O(n)。这是顺序表插入操作的核心考点。

### 13. 通过折半查找对关键字序列 {12, 18, 21, 35, 45, 55, 66}，查找 21 需要查找（ ）次

A. 4　　B. 3　　C. 2　　D. 1

> **答案：B**
>
> **解析：** 折半查找过程（序列已升序排列）：
> - 第1次：mid = (0+6)/2 = 3，`a[3]=35`，21 < 35，去左半边 {12,18,21}
> - 第2次：mid = (0+2)/2 = 1，`a[1]=18`，21 > 18，去右半边 {21}
> - 第3次：mid = (2+2)/2 = 2，`a[2]=21`，找到！
>
> 共 3 次查找。

### 14. 串 `"abcd"` 的长度为（ ）

A. 5　　B. 2　　C. 6　　D. 4

> **答案：D**
>
> **解析：** 串的长度是指**串中字符的个数**，不含结束符 `\0`。`"abcd"` 有 4 个字符，所以长度为 4。区分：`sizeof("abcd")` 返回 5（含 `\0`），`strlen("abcd")` 返回 4。

### 15. 数据的结构类型，从逻辑上可分为（ ）

A. 动态结构和静态结构
B. 紧凑结构和非紧凑结构
C. 线性结构和非线性结构
D. 内部结构和外部结构

> **答案：C**
>
> **解析：** 数据结构从两个角度分类：
> - **逻辑结构**：线性结构（线性表、栈、队列、串）vs 非线性结构（树、图、集合）
> - **存储结构**：顺序存储、链式存储、索引存储、散列存储
>
> A/B/D 都不是数据结构的标准分类。

### 16. 链式存储的存储结构所占空间（ ）

A. 只有一部分存放结点值
B. 只有一部分存放结点间关系的指针
C. 分两部分，一部分存放结点值，另一部分存放结点所占单元数
D. 分两部分，一部分存放结点值，另一部分存放表示结点间关系的指针

> **答案：D**
>
> **解析：** 链表每个结点包含**两个部分**：数据域（结点值）+ 指针域（指向后继结点的指针）。C 错在"结点所占单元数"——指针域存的是相邻结点的地址，而非所占单元数。

### 17. 带有头结点的单链表，头指针为 `head`，则表示单链表为空的选项是（ ）

A. `head == NULL`
B. `head != NULL`
C. `head->next == NULL`
D. `head->next == head`

> **答案：C**
>
> **解析：** 带头结点的单链表，头结点始终存在（即使链表为空）。判断空的标志是**头结点后面没有实际数据结点**，即 `head->next == NULL`。A 是不带头结点的判空方式。D 是循环链表的判空方式。

### 18. 四个元素 1、2、3、4 依次入栈，第一个出栈元素为 4，则第 3 个出栈元素为（ ）

A. 1　　B. 2　　C. 3　　D. 4

> **答案：B**
>
> **解析：** 栈是后进先出（LIFO）。1、2、3、4 依次入栈，第一个出栈的是 4，说明四个元素全部入栈后才开始弹出：
> - Push 1, 2, 3, 4 → 栈内：`[1, 2, 3, 4]`（4在栈顶）
> - Pop → 4（第1个出栈）
> - Pop → 3（第2个出栈）
> - Pop → 2（**第3个出栈**）
> - Pop → 1（第4个出栈）

---

### 19. 二叉树第 3 层的结点个数最多有（ ）个

A. 1　　B. 4　　C. 8　　D. 16

> **答案：B**
>
> **解析：** 二叉树的第 k 层最多有 `2^(k-1)` 个结点（满二叉树）。第 3 层：2^(3-1) = 2² = 4 个。（注意第 1 层从 1 开始计数，根结点在第 1 层。）

---

### 20. 在单链表中，要将 s 所指向结点插入到 p 所指向结点之后，其语句应为（ ）

A. `s->next = p + 1; p->next = s;`
B. `s->next = p->next; p->next = s->next;`
C. `(*p).next->s; (*s).next = (*p).next;`
D. `s->next = p->next; p->next = s;`

> **答案：D**
>
> **解析：** 单链表插入的标准两步操作：
> 1. 先让新结点 `s` 指向 `p` 的后继：`s->next = p->next`
> 2. 再让 `p` 指向新结点 `s`：`p->next = s`
>
> **顺序不能反！** 如果先执行 `p->next = s`，`p` 原来的后继结点就丢失了。
>
> B 错在第二步 `p->next = s->next`（等于把 p 的后继设成了自己）。A 中 `p+1` 是地址运算，毫无意义。C 语法错误。

## 二、判断题（本大题共10小题，每小题2分，共20分）

判断以下说法是否正确，正确填"√"，错误填"×"。

### 1. 结构体变量的成员可以通过 `.成员名` 来引用其值。（ ）

> **答案：√**
>
> **解析：** 结构体变量的成员通过 `.` 运算符访问，如 `stu.name`。若是指向结构体的指针，则需用 `->` 运算符或 `(*p).成员名`。

---

### 2. 数组的下标从 1 开始。（ ）

> **答案：×**
>
> **解析：** C语言中数组下标从 **0** 开始。`int a[5]` 的合法下标是 `a[0]` 到 `a[4]`。

---

### 3. 复合语句（`{}`）中定义的变量，其作用域仅限于该复合语句内部。（ ）

> **答案：√**
>
> **解析：** 在花括号 `{}` 内部定义的变量为**局部变量**，作用域从定义处开始到该 `}` 结束。离开复合语句后变量被释放，外部不可访问。

---

### 4. 不同函数中可以使用同名的变量，它们互不影响。（ ）

> **答案：√**
>
> **解析：** 不同函数中的局部变量各自独立，存储在各自的栈帧中，即便同名也互不影响。这也是信息隐藏的基础。

---

### 5. 栈和队列没有共同点。（ ）

> **答案：×**
>
> **解析：** 栈和队列都是**操作受限的线性表**，这是它们的共同点。区别在于：栈是 LIFO（后进先出），队列是 FIFO（先进先出）。

---

### 6. `while` 语句的退出方式只有条件表达式为假这一种。（ ）

> **答案：×**
>
> **解析：** 除了条件为假正常退出外，还可以通过 `break` 语句强制跳出循环。当然，`return` 或 `exit()` 也可以提前结束循环所在的整个函数。

---

### 7. 二维数组在内存中按行优先顺序存储。（ ）

> **答案：√**
>
> **解析：** C语言中，二维数组按**行优先**存储。例如 `int a[2][3]` 在内存中的顺序是：`a[0][0], a[0][1], a[0][2], a[1][0], a[1][1], a[1][2]`。这解释了为什么可以用一个指针遍历整个二维数组。

---

### 8. 串是一种特殊的线性表，其数据元素只能是字符。（ ）

> **答案：√**
>
> **解析：** 串（字符串）的定义就是**零个或多个字符组成的有限序列**，本质上是一种数据元素限定为字符的线性表。

---

### 9. 算法的时间复杂度和空间复杂度一定呈反比关系。（ ）

> **答案：×**
>
> **解析：** 时间复杂度和空间复杂度没有必然的反比关系。有些算法同时具有较好的时间和空间效率（如原地排序），有些则两者都差。只有某些特定场景下才存在"以空间换时间"的权衡。

---

### 10. 在线性表中，除了第一个元素外，每个元素有且只有一个直接前驱。（ ）

> **答案：√**
>
> **解析：** 线性表的定义特性：除第一个元素（无前驱）和最后一个元素（无后继），其他每个元素**有且只有一个直接前驱和一个直接后继**。

## 三、填空题（本大题共5小题，每小题4分，共20分）

### 1. 下面程序的功能是通过指针形参返回两个数的和，请补全。

```c
void add(int x, int y, int *z) {
    ________;
}
```

> **答案：** `*z = x + y`
>
> **解析：** `z` 是指向 `int` 的指针，要通过指针修改外部变量的值，需用 `*z` 解引用。`*z = x + y` 将两数之和写入指针指向的地址。

---

### 2. 执行以下程序段后，输出结果是 `____`。

```c
char s1[20] = "Hello";
char s2[20] = "Hi";
printf("%d", strlen(strcpy(s1, s2)));
```

> **答案：** 2
>
> **解析：** `strcpy(s1, s2)` 将 `s2`（`"Hi"`）复制到 `s1`，返回 `s1` 的地址。`strlen("Hi")` = 2。注意：复制后 `s1` 的内容变为 `"Hi"`（含 `\0`），长度是 2 不是 5。

---

### 3. 串的两种基本存储方式是：`____` 和 `____`。

> **答案：** 顺序存储、链式存储
>
> **解析：** 串和线性表一样，有两种基本存储方式。顺序存储用字符数组（连续内存），链式存储用字符链表。实际中最常用的是顺序存储。

---

### 4. 广义表 `L = (a, (b, c), ())` 的长度是 `____`。

> **答案：** 3
>
> **解析：** 广义表的**长度**是指第一层元素的个数。L 的第一层有三个元素：原子 `a`、子表 `(b, c)`、空表 `()`。所以长度为 3。注意区分广义表的**深度**（括号的最大嵌套层数），L 的深度是 2。

---

### 5. 9 个顶点的连通图最少需要 `____` 条边。

> **答案：** 8
>
> **解析：** n 个顶点的连通图最少需要 `n - 1` 条边，此时形成一棵树（无环连通图）。9 个顶点最少需要 8 条边。若少于 8 条边，图必不连通。

## 四、简答题（本大题共4小题，共 40 分）

### 1. 什么是数据项？什么是抽象数据类型（ADT）？（10分）

> **参考答案：**
>
> **数据项**：是数据结构中讨论的**最小单位**，不可再分割。例如描述一个学生时，学号、姓名、成绩各自都是数据项。
>
> **抽象数据类型（ADT）**：是指一个数学模型以及定义在该模型上的一组操作。它有两个特征：
> - **抽象性**：只关心"做什么"（逻辑特性），不关心"怎么做"（物理实现）
> - **封装性**：数据和操作封装在一起，外部只能通过定义好的接口访问
>
> ADT 通常用三元组表示：**ADT = (D, S, P)**，其中 D 是数据对象，S 是 D 上的关系集，P 是操作集。
>
> **简例**：整数类型的 ADT 包含数据 `{..., -1, 0, 1, ...}` 和操作 `{+, -, *, /, %}`，用户使用整数运算时不需要知道底层二进制如何实现。

### 2. 分析以下程序段的功能，并写出循环结束时 i 的值。（10分）

```c
int a[10] = {85, 72, 93, 68, 80, 76, 90, 83, 78, 88};
int sum = 0, i;
for (i = 0; i < 10; i++) {
    sum += a[i];
}
printf("平均值 = %.1f\n", sum / 10.0);
```

> **参考答案：**
>
> **程序功能**：计算数组 `a` 中 10 个元素的**平均值**并输出。
>
> **循环结束时 i = 10**。因为当 `i = 10` 时条件 `i < 10` 不成立，循环退出。此时 `sum` 是所有元素之和（803），输出 `平均值 = 80.3`。

---

### 3. 简述顺序表存储结构的优缺点。（10分）

> **参考答案：**
>
> **优点：**
> 1. **随机访问**：可通过下标 O(1) 访问任意元素
> 2. **存储密度高**：只存数据，没有指针开销，空间利用率高
> 3. **实现简单**：用数组即可实现，逻辑清晰
>
> **缺点：**
> 1. **插入/删除效率低**：需要移动大量元素，平均 O(n)
> 2. **空间固定**：需要预先分配最大容量，容易浪费或不够用
> 3. **连续空间要求**：对内存碎片敏感，大块连续空间可能分配失败
>
> **适用场景**：数据量已知、频繁查找而少做插入删除的情况。

---

### 4. 什么是有向图？什么是有向图中顶点的出度和入度？（10分）

> **参考答案：**
>
> **有向图**：由顶点集和边集组成，其中每条边都是一个**有序对** `<v, w>`，表示从顶点 v 指向顶点 w 的弧（有方向的边）。`<v, w>` 和 `<w, v>` 在图中是不同的边。
>
> **出度（Out-degree）**：以顶点 v 为**弧尾**的弧的数目，即从 v 出发指向其他顶点的边数。记作 `OD(v)`。
>
> **入度（In-degree）**：以顶点 v 为**弧头**的弧的数目，即从其他顶点指向 v 的边数。记作 `ID(v)`。
>
> **性质**：一个有向图中，所有顶点的出度之和 = 所有顶点的入度之和 = 图中的边数。

## 五、计算题（本大题共3小题，每小题10分，共30分）

### 1. 写出以下程序的输出结果。

```c
#include <stdio.h>

int main() {
    int x;
    scanf("%d", &x);      // 假设输入 75
    int n;
    switch (x / 10) {
        case 0:
        case 1:
        case 2:
        case 3:
        case 4:
        case 5: n = 0; break;
        case 6:
        case 7: n = 1; break;
        case 8: n = 2; break;
        case 9:
        case 10: n = 3; break;
        default: n = -1;
    }
    printf("n = %d\n", n);
    return 0;
}
```

> **答案：** `n = 1`
>
> **解析：** 输入 75，`x / 10 = 7`。匹配 `case 7`，执行 `n = 1`，然后 `break` 跳出。该程序的功能相当于**将百分制成绩映射为等级**：0-59→0（不及格），60-79→1（中等），80-89→2（良好），90-100→3（优秀）。

### 2. 写出以下程序的输出结果。

```c
#include <stdio.h>

int main() {
    int i = 153;
    int a, b;
    a = i / 100;           // 百位
    b = (i / 10) % 10;     // 十位
    printf("a = %d, b = %d\n", a, b);
    return 0;
}
```

> **答案：** `a = 1, b = 5`
>
> **解析：** 程序将三位数 153 分解各位数字：
> - 百位：`153 / 100 = 1`（整数除法截断）
> - 十位：`(153 / 10) % 10 = 15 % 10 = 5`
> - 个位（未输出）：`153 % 10 = 3`
>
> 这是经典的**数位分解**套路，竞赛和考试经常出现。

---

### 3. 写出以下程序的输出结果。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char s[] = "abcdefgh";
    char t[20];
    int j = 0;
    for (int i = 0; s[i] != '\0'; i++) {
        if (s[i] != 'c') {
            t[j] = s[i];
            j++;
        }
    }
    t[j] = '\0';
    printf("%d\n", j);
    printf("%s\n", t);
    return 0;
}
```

> **答案：**
> - 第一行输出：`7`
> - 第二行输出：`abdefgh`
>
> **解析：** 程序遍历字符串 `s`，将**不等于 'c'** 的字符复制到 `t` 中。`s` 原为 `"abcdefgh"`（共 8 个字符），去掉 `'c'` 后 `t` = `"abdefgh"`（共 7 个字符）。`j` 既是下标计数器，也是新字符串的长度。

### 3. 编程：计算 1~300 之间所有奇数的和与偶数的和，并分别输出。

> **参考答案：**
> ```c
> #include <stdio.h>
>
> int main() {
>     int sum_odd = 0;   // 奇数和
>     int sum_even = 0;  // 偶数和
>
>     for (int i = 1; i <= 300; i++) {
>         if (i % 2 == 1) {
>             sum_odd += i;    // 奇数
>         } else {
>             sum_even += i;   // 偶数
>         }
>     }
>
>     printf("奇数和 = %d\n", sum_odd);
>     printf("偶数和 = %d\n", sum_even);
>     return 0;
> }
> ```
>
> **输出：**
> ```
> 奇数和 = 22500
> 偶数和 = 22650
> ```
>
> **验证**：1~300 总和 = `300×301/2 = 45150`，奇数和 + 偶数和 = `22500 + 22650 = 45150` ✅
>
> **评分要点**：变量初始化正确（2分）| 循环范围正确（2分）| 奇偶判断逻辑正确（3分）| 累加逻辑正确（2分）| 输出正确（1分）
>
> **优化写法**：利用等差数列公式，奇数和 = `150×(1+299)/2 = 150×150 = 22500`，偶数和 = `150×(2+300)/2 = 150×151 = 22650`，O(1) 直接出结果。
